## 🚀 Alur Kontribusi

Ikuti langkah-langkah ini untuk mulai berkontribusi:

1.  **Fork**: Buat salinan repository ini ke akun GitHub Anda.
2.  **Branch**: Buat branch baru untuk fitur atau perbaikan Anda.
    ```bash
    git checkout -b feature/nama-fitur-anda
    ```
3.  **Develop**: Tulis kode Anda (jangan lupa cek [Development Guide](DEVELOPMENT.md)).
4.  **Test**: Jalankan `npm run lint` dan `npm run build`.
5.  **Commit**: Gunakan format commit yang benar.
6.  **PR**: Kirim Pull Request ke branch `main` repository utama.

## 📝 Aturan Commit Message

Kami mengikuti standar **Conventional Commits** untuk memudahkan pembuatan changelog otomatis.

| Tipe | Deskripsi | Contoh |
| :--- | :--- | :--- |
| `feat` | Fitur baru | `feat: implementasi dark mode` |
| `fix` | Perbaikan bug | `fix: perbaiki error login` |
| `docs` | Perubahan dokumentasi | `docs: perbarui panduan setup` |
| `style` | Perubahan UI/Formatting | `style: perbaiki margin footer` |
| `perf` | Optimasi performa | `perf: percepat loading gambar` |

## 🏗️ Standar Kualitas

Agar kode tetap terjaga kualitasnya, mohon perhatikan hal berikut:

- **TypeScript**: Gunakan pengetikan yang kuat. Hindari penggunaan `any` sebisa mungkin.
- **Clean Code**: Tulis fungsi yang kecil dan deskriptif.
- **Accessibility**: Pastikan elemen UI memiliki atribut ARIA yang tepat.
- **Responsive**: Tes fitur Anda di berbagai ukuran layar.

## 📮 Melaporkan Masalah

Jika Anda menemukan bug tetapi tidak sempat memperbaikinya, silakan buka **Issue** baru. Pastikan Anda menyertakan:
- Deskripsi masalah yang jelas.
- Langkah-langkah untuk mereproduksi bug.
- Screenshot atau log error jika ada.
